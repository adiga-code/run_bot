Telegram-ботаКлючевая логика MVP Telegram-бота.Важно:максимально простая реализация, без сложной аналитики, но с защитой пользователя от перегрузки и травм.1. Общая логика- Все тренировки заранее загружены (28 дней × уровни)- Программа фиксированная и НЕ изменяется- Бот НЕ генерирует тренировки- Бот только:  — показывает тренировку дня  — выбирает версию: base / light / recovery  — отслеживает выполнение2. Структура тренировокДля каждого дня есть:- тип: бег / силовая / восстановление / отдых- 3 версии: base / light / recoveryСиловые:- отображаются только как:  &quot;Силовая тренировка 1&quot;- и далее открыть тренировку   &quot;Силовая тренировка 2&quot; и далее открыть тренировку- упражнения не показываются3. Утренний чек-инКаждый день бот задаёт:1) Как ты себя чувствуешь?   отлично / нормально / тяжеловато / плохо2) Как спал?   хорошо / нормально / плохо3) Есть ли боль?   нет / немного / есть4) Был ли сильный внешний стресс за последние 24 часа?   нет / да умеренный / да сильный---4. Логика выбора тренировкиСчитается простой “стресс-балл”:- тяжеловато = +1- плохо = +2- сон плохо = +1- боль немного = +2- боль есть = сразу recovery- внешний стресс умеренный = +1- внешний стресс сильный = +2Результат:0–1 → Base  2–3 → Light  4+ → Recovery  Если есть боль → всегда Recovery или отдых---5. Дополнительная защитаЕсли 2 дня подряд:- тяжеловато / плохо- или был стресс- или тренировки выполнены частично/пропущены→ на 3-й день автоматически ставится Light или Recovery---6. Ограничения нагрузкиПосле силовой:- нельзя давать тяжёлый бег- только лёгкий бег / восстановление / отдых---7. Отметка выполненияПользователь отмечает:- выполнено- частично- пропущено---8. Логика прогресса по неделямВ конце каждой недели рассчитывается процент выполнения:- выполнено = 100%- частично = 50%- пропущено = 0%Если ≥ 75%:— открывается следующая неделяЕсли &lt; 75%:— следующая неделя НЕ открывается— текущая неделя повторяется полностью— повторяются все дни без изменений---9. Важно- программа НЕ пересобирается- тренировки НЕ изменяются- используется та же неделя повторно- прогресс внутри недели обнуляется при повторе- история может сохраняться (опционально)---10. Ограничение адаптацииБот НЕ адаптирует программу.Адаптация происходит только через:— выбор версии дня (base / light / recovery)---11. Красные флаги (обязательно)Если:- есть боль- боль усиливается- состояние &quot;плохо&quot; + стресс→ только Recovery или отдых  → никакой нагрузки---12. Принцип MVPГлавное:- простота- безопасность- понятностьЕсли есть сомнение:→ всегда выбирается более лёгкий вариант
